### 2. ✅ 模型列表同步（任务 #2）
- **后端**：`GET /api/gateway/models` 用当前网关 Base URL + Key 请求上游 `/v1/models`，返回模型 ID 数组

### 3. ✅ 高级网络设置（任务 #3）
- **前端**：网关面板下的折叠区域，包含：

- **测试**：验证自定义头 `X-Relay-Test: hello` 被转发到 fake upstream（test/features.test.js）

### 4. ✅ 实时调试日志面板（任务 #4）
- **后端**：`lib/debugLog.js` 内存环形缓冲（500 条），`addLog()` 记录每次网关转发（timestamp / method / url / status / duration_ms / error_message / model / mappedTo），脱敏 Authorization 和 api_key（仅保留首尾各 4 位）

- **测试**：验证日志捕获、字段完整、密钥不泄露（test/features.test.js）

### 5. ✅ 配置历史与回滚（任务 #5）
- **后端**：
  - `GET /api/config/history?product=` 读取对应产品的 backups/ 目录，返回 `{product, configPath, backups: [{filename, mtime}]}`
  - `POST /api/config/rollback` 接收 `{product, filename}`，原子写入当前配置前先备份，写完后重启对应客户端（`restartApp`）
  - 路径穿越防护：`path.resolve()` 确认备份文件在 backups/ 目录内
- **前端**：每个产品面板旁的「🕑 历史版本」按钮，弹出模态框列表，每条带「回滚」按钮
- **测试**：验证 API 返回 backups 数组（test/features.test.js）

### 6. ✅ 进程状态与重启优化（任务 #6）
- **后端**：
  - `GET /api/process/status?product=` 返回 `{running, pid, name, product}`（跨平台：tasklist/pgrep）

- **测试**：验证 API 返回 `{running: boolean}` 形状（test/features.test.js）

### 7. ✅ 测试连接增强（任务 #7）

## 二、技术实现要点

### 后端架构（`server.js`）
- **模块化辅助**：新增 `lib/debugLog.js`（日志环 + 脱敏逻辑）
- **网关转发重构**：`forwardToUpstream` 包裹 `attempt()` 内层递归实现重试，每次读 `cfg.network` 热加载配置，`sentToClient` 标记防止重试已发送的响应
- **helper 函数**：
  - `getProcessStatus(product)` / `findFirstPid(name)` — 跨平台进程检测
  - `listConfigHistory(product)` / `rollbackConfig(product, filename)` — 备份列表/回滚
  - `productToConfigPath(product)` — 产品配置文件路径映射

### 前端架构（`index.html`）
- **状态管理**：全局 `state`、`gwRoutes`、`netHeaders`，通过 `loadState()` / `loadGateway()` 从 API 拉取并填充表单
- **面板切换**：`switchPanel(name)` 切 `.active` class
- **动态渲染**：`renderRoutes()` / `renderHeaders()` / `renderLogs()` / `openHistory()` 读 JS 数组生成 DOM
- **轮询机制**：日志面板每 2s 调 `pollLogs()`；进程状态每 8s 调 `refreshProcStatus()`
- **校验**：`requireFields(ids)` 遍历字段检查非空，标记 `.invalid` class

### 测试体系
- **test/toml.test.js**（9 项）：Codex CLI TOML 往返测试（@iarna/toml 正确处理嵌套表/布尔/转义/malformed）
- **test/gateway.test.js**（11 项）：网关集成测试（模型改写、thinking 注入/预算、auth 头、错误归一化、429 透传）

## 五、安全 & 健壮性

### 安全加固
1. **日志脱敏**：Authorization / x-api-key / token 等字段仅保留首尾各 4 位（`redactSecret` / `redactAuthValue`）
2. **路径穿越防护**：`rollbackConfig` 用 `path.resolve()` 确认备份文件在 backups/ 内
3. **必填字段校验**：前端阻止空 Base URL / API Key 保存，后端仍应增加二次校验（当前前端已充分阻止）
4. **Host 检查**：仍保持 127.0.0.1 绑定 + Host 头校验（已有）

### 健壮性增强
1. **网关重试**：连接级错误自动重试（maxRetries 次），HTTP 错误不重试（已发送响应头）
2. **超时控制**：请求超时/读取超时可配置，0 = 使用默认（流式 10 分钟 / 非流式 2 分钟）
3. **错误归一化**：所有上游错误转为 Anthropic `{type:'error',error:{type,message}}` 格式
4. **进程检测跨平台**：Windows tasklist / macOS-Linux pgrep，未检测到返回 `running: false`

## 六、已知限制 & 注意事项

### 工具侧问题
- **文件截断**：本次开发中多次遇到 Write 工具在写入 server.js / index.html 等大文件时**尾部被截断**（缺少闭合标签/半行/NUL 字节）。每次大改后必须 `node --check` 验证；被截断时用 shell `head -n N` + 手工补完文件尾，或改用 `cat >>` heredoc 分段写入。已在 memory 中记录此问题。

### 功能限制
- **前端无构建**：遵循约束，仍为单页 HTML 无打包，CSS/JS 内联；适合轻量管理工具，不适合大型 SPA。
- **配置回滚范围**：仅限已备份的配置文件（每次保存前自动备份，保留最近 20 个）；若用户手动删除 backups/ 目录则无历史版本。
- **进程重启依赖 exe 路径**：若 `paths.json` 未配置或自动检测失败，重启只能 kill 无法 relaunch（前端会显示警告让用户手动重开）。
