🛣 中转网关（模型名改写代理）

RelayManager 内置一个本地网关，替代 CC Switch 的网关。Claude Desktop 把请求发到本地网关，网关把 Anthropic 模型名（如 claude-sonnet-4-5）改写成后端模型（如 glm-5.2），再转发给真实中转站。这样无需 CC Switch，Claude Desktop 也能用没有 Claude 模型的中转站。

💡 原理：Claude Desktop → 本地网关（改写模型名）→ 真实中转站。网关随 RelayManager 启动，需保持 RelayManager 运行。

左栏填 Claude Desktop 要求的 Anthropic 模型名（如 claude-sonnet-4-5），右栏填中转站实际支持的后端模型（如 glm-5.2，可从「模型映射」标签页获取后选）。Claude Desktop 发左栏名，网关改写成右栏名转发。
